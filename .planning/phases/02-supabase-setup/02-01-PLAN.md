**File 4: 20260309000004_custom_access_token_hook.sql** — Create the Custom Access Token Hook function exactly per research:

```sql
CREATE OR REPLACE FUNCTION public.custom_access_token_hook(event jsonb)
RETURNS jsonb LANGUAGE plpgsql STABLE AS $$
DECLARE
  claims jsonb;
  is_admin boolean;
BEGIN
  claims := event->'claims';
  SELECT COALESCE((raw_app_meta_data->>'is_admin')::boolean, false)
  INTO is_admin FROM auth.users
  WHERE id = (event->>'user_id')::uuid;
  claims := jsonb_set(claims, '{is_admin}', to_jsonb(is_admin));
  event := jsonb_set(event, '{claims}', claims);
  RETURN event;
END;
$$;
```

Include all required GRANTs:
- GRANT USAGE ON SCHEMA public TO supabase_auth_admin
- GRANT EXECUTE ON FUNCTION public.custom_access_token_hook TO supabase_auth_admin
- REVOKE EXECUTE ON FUNCTION public.custom_access_token_hook FROM authenticated, anon, public
- GRANT SELECT ON TABLE auth.users TO supabase_auth_admin
